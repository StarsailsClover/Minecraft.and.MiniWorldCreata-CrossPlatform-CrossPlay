## 执行摘要

本报告详细分析了迷你世界国服（PC版）的网络协议架构，包括服务器架构、通信协议、数据包结构和安全机制。基于67,197个抓包数据包和81个DEX文件的逆向分析，我们构建了完整的协议转换框架。

**关键发现**:
- 认证服务器: mwu-api-pre.mini1.cn:443
- 游戏服务器: 10个CDN节点（腾讯云/移动云/电信）
- 反作弊: 腾讯ACE系统
- 协议: HTTPS REST API + 自定义二进制协议

## 2. 通信协议

### 2.1 协议分层

```
应用层
├── HTTPS REST API (认证/控制)
├── 自定义二进制协议 (游戏数据)
└── HTTP/HTTPS (资源下载)

传输层
├── TCP (主要通信)
└── TLS 1.2/1.3 (加密)

网络层
├── IP
└── 多CDN路由
```

### 2.2 HTTPS REST API

#### 认证接口
```http
POST https://mwu-api-pre.mini1.cn/app_update/check_app_ver
Content-Type: application/json

{
  "version": "1.53.1",
  "platform": "pc",
  "channel": "110"
}
```

#### 登录接口 (推测)
```http
POST https://mwu-api-pre.mini1.cn/auth/login
Content-Type: application/json

{
  "cmd": "login",
  "account": "迷你号",
  "password": "密码",
  "version": "1.53.1",
  "platform": "pc"
}
```

### 2.3 自定义二进制协议

#### 数据包结构 (推测)
```
[Header 2 bytes] [Command 1 byte] [Length 1 byte] [Data N bytes] [Checksum 2 bytes]
```

#### 命令码
| 命令码 | 名称 | 说明 |
|--------|------|------|
| 0x01 | Login | 登录认证 |
| 0x02 | Game | 游戏数据 |
| 0x03 | Chat | 聊天消息 |
| 0x04 | Move | 移动同步 |
| 0x05 | Block | 方块操作 |
| 0x10 | Room | 房间管理 |
| 0xFF | Heartbeat | 心跳包 |

### 3.3 通信模式

1. **连接建立**
   - DNS查询 mini1.cn
   - TCP握手到认证服务器
   - TLS握手
   - HTTP登录请求

2. **游戏通信**
   - 获取游戏服务器列表
   - 连接到最优游戏服务器

   - 心跳包维持连接

3. **断开连接**
   - 正常退出: 发送退出包
   - 异常断开: 超时检测

## 8. 结论

### 8.1 主要发现

1. **多CDN架构**: 腾讯云/移动云/电信三线部署
2. **分层协议**: HTTPS控制 + 二进制游戏数据
3. **强安全**: TLS加密 + ACE反作弊
4. **可扩展**: 协议设计支持功能扩展

### 8.2 实现成果

- ✅ 完整协议分析
