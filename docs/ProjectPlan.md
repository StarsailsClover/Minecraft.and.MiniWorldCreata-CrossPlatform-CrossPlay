# 项目规划与开发计划文档

本文档包含项目背景、开发计划、风险分析和研究规划等非技术实现细节。

---

## 1. 项目背景：初中生原型方案回顾

### 1.1 已暴露问题 + 对应解决方案

| 序号 | 已暴露问题 | 技术根因 | 解决方案 |
|------|------------|----------|----------|
| 1 | 仅支持迷你国服手游 ↔ MC基岩版，不支持Java/PC/外服 | 直接用gophertunnel做**迷你→基岩直连**，无Java中转层 | 采用「迷你四端 → Java中转 → Geyser → 基岩版」架构，一次开发兼容MC全版本 |
| 2 | 无法联动Java版，Java→基岩模组仅支持单向 | 未利用成熟的GeyserMC生态，路径设计错误 | 1. 先开发「迷你→Java协议桥」<br>2. 接入GeyserMC自动转发至基岩版<br>3. 复用Java生态所有兼容能力 |
| 3 | 迷你手游(6人) / PC(40人) 端差异无法适配 | 仅适配手游协议，未处理PC端数据包、房间上限、同步频率 | 开发**端差异适配层**：<br>• 手游：限制房间人数、精简数据包<br>• PC：开放40人、完整同步逻辑<br>• 代理层统一封装为Java标准协议 |
| 4 | 仅支持国服，外服MiniWorld: Creata不兼容 | 未区分国服/外服登录、加密、方块ID | 开发**国服/外服独立适配模块**：<br>• 登录：国服迷你号 / 外服Google/FB OAuth<br>• 加密：两套加解密算法<br>• 方块/实体：双版本ID映射表 |
| 5 | 仅实现基础移动/挖掘/聊天，功能残缺 | 仅做最小原型，未处理核心玩法同步 | 完善**数据包翻译层**：覆盖方块放置、背包、合成、掉落物、生物同步 |
| 6 | 方块镜像(X轴颠倒)显示异常 | 迷你与MC坐标系定义相反 | 在协议桥中**强制X轴坐标取反**，实时修正坐标数据 |

### 1.2 潜在未发现问题 + 预判解决方案

| 序号 | 潜在未发现问题 | 风险影响 | 预判解决方案 |
|------|----------------|----------|--------------|
| 1 | 迷你官方协议加密/签名更新，原型瞬间失效 | 联机直接断开、无法登录 | 1. 协议层做**热更新配置**，加密算法可动态替换<br>2. 本地缓存协议规则，支持快速适配 |
| 2 | 官方SAC反作弊系统检测到第三方代理 | 客户端闪退、账号封禁 | 1. 代理层**模拟官方客户端心跳/校验包**<br>2. 不修改客户端本体，仅做流量转发<br>3. 屏蔽异常行为上报 |
| 3 | 多人联机时数据包丢包、延迟不同步 | 卡方块、位置漂移、操作不同步 | 1. 加入**延迟补偿 + 帧插值**算法<br>2. 关键操作包重发机制<br>3. 区分实时包/非实时包优先级 |
| 4 | 迷你与MC实体ID、方块ID完全不匹配 | 方块显示错误、生物消失、道具异常 | 构建**全量ID映射库**：<br>• 方块ID一一映射<br>• 生物/道具/粒子效果适配<br>• 国服/外服双映射表 |
| 5 | 多端数据冲突（手游存档/PC存档不互通） | 进服闪退、地图错乱 | 代理层**统一存档格式**，屏蔽客户端原生存档差异，仅同步实时操作数据 |
| 6 | MC红石/指令/命令方块无法兼容 | 红石失效、指令无响应 | 1. 基础版：屏蔽红石/指令系统<br>2. 进阶版：迷你简易逻辑 → MC红石指令翻译 |
| 7 | 性能瓶颈（高并发房间卡顿） | 多人联机掉帧、延迟飙升 | 1. 服务端采用PaperMC异步处理<br>2. 数据包压缩、减少冗余传输<br>3. 分房间独立进程隔离 |
| 8 | 版本迭代后兼容失效（迷你/MC更新） | 新方块/新功能无法同步 | 设计**版本适配配置文件**，无需重构代码，仅更新映射表与协议规则 |
| 9 | 海外版网络延迟、区域限制 | 外服玩家无法连接、高延迟 | 1. 代理服务支持**国内外节点分流**<br>2. 外服专用UDP加速通道 |
| 10 | 离线模式/盗版客户端兼容问题 | 部分玩家无法进服 | 支持**离线/在线双模式验证**，适配私服/官方客户端 |

## 2. 标准化全流程开发步骤

### 阶段1：开发准备（1-3天）
1. 版本锁定：迷你国服1.53.1、外服1.7.15、MC Java 1.20.6
2. 工具链部署：Wireshark、Frida、APKTool、IDA、gophertunnel
3. 环境搭建：本地PaperMC + Fabric + GeyserMC + Floodgate
4. 测试账号：迷你国服/外服、MC Java/基岩版各2个

### 阶段2：四端协议逆向与抓包（3-7天）
1. 抓取全量数据包：登录、进房、移动、挖掘、放置、聊天、合成
2. 破解加密算法：国服AES-128、外服AES-256
3. 解析数据包结构：字段含义、顺序、校验规则
4. 输出《迷你四端协议差异说明书》

### 阶段3：端适配层开发（3-5天）
1. 客户端类型自动识别模块
2. 国服/外服双加解密实现
3. 手游6人/PC40人房间限制逻辑
4. 数据包裁剪/补全工具函数

### 阶段4：核心协议翻译层开发（7-14天）
1. 迷你数据包 ↔ MC Java数据包解析/构建
2. 坐标X轴取反模块（解决方块镜像）
3. 国服/外服方块/实体ID映射表
4. 基础操作（移动/挖掘/聊天）翻译

### 阶段5：Java中转 + Geyser对接（3-5天）
1. 代理服务器连接PaperMC
2. 集成GeyserMC + Floodgate
3. 基岩版免账号登录适配
4. 双MC版本互通验证

### 阶段6：全端联机功能补齐（7-14天）
1. 背包、合成、掉落物同步
2. 生物、载具、粒子效果适配
3. 多人并发、延迟补偿、丢包重传
4. SAC反作弊规避（不修改客户端，纯流量转发）

### 阶段7：多端联机型测试（3-7天）
1. 四端单独进服测试
2. 四端混合联机测试
3. 弱网/高延迟场景测试
4. 6人/40人房间压力测试

### 阶段8：优化与迭代（长期）
1. 性能优化：内存占用、CPU占用、网络流量
2. 热更新配置：协议变更、ID映射更新
3. 文档维护：协议文档、部署文档、使用说明

## 3. 核心模块深入研究方向

### 3.1 协议逆向自动化
- 自动抓包、自动解析数据包结构
- 加密算法自动识别、动态更新

### 3.2 全量ID映射库
- 方块/实体/物品/粒子/成就全映射
- 支持可视化配置、一键更新

### 3.3 同步与网络优化
- 帧同步 + 状态同步混合方案
- 弱网环境抗丢包、延迟补偿算法

### 3.4 反作弊与安全
- 官方客户端行为模拟
- 流量伪装、无侵入式代理

### 3.5 跨版本兼容
- 迷你多版本、MC多版本向前兼容
- 配置化适配，无需重构代码

## 4. 风险控制与合规说明

### 4.1 法律风险
- 本方案**仅用于技术研究与学习**，禁止商业运营
- 不破解游戏本体、不盗用资源、不传播私服

### 4.2 官方封禁风险
- 仅做流量转发，不修改客户端
- 预留紧急开关，可快速停止服务

### 4.3 技术风险
- 协议变更：采用热更新配置快速适配
- 性能问题：分布式部署、分房间隔离
